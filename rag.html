RAG Flow

1. Source Input

Provide raw documents or information that will act as the retrieval knowledge base.

2. Chunking

Split the document into smaller chunks using chunk size and overlap.

3. Embeddings

Each chunk gets converted into vector embeddings for semantic search.

4. Vector Storage

Store vector + original chunk + metadata into vector database.

5. Retrieval

User query is converted into embeddings and similar chunks are retrieved.

6. Augmentation

Relevant chunks are injected into the prompt sent to the LLM.
